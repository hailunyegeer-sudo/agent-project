# Agent 运行记录与工具调用日志

## 文档目的

本文件刻意用于制造“关键词分别出现，但没有完整短语”的检索场景。它会提到 Agent，也会提到工具调用，但不会连续写出“Agent工具调用”或“Agent 工具调用”。因此，当查询为 `Agent 工具调用` 时，本文件应该被召回，却不应该排在精确短语文档前面。

## Agent 运行记录

Agent 的运行由 B1 负责。B1 会构造 SystemMessage、HumanMessage、AIMessage 与 ToolMessage 的完整消息序列。每次模型输出后，B1 判断本轮是否出现 tool_calls。若模型只是给出普通回答，则流程结束；若模型需要外部信息，B1 将请求交给 B3。

## 工具调用日志

工具调用日志的内容通常包括调用时间、工具名称、输入参数、执行状态、错误信息与耗时。B3 会读取 tools.yaml 中的配置，确认工具属于当前 toolset，然后动态导入 B2 中的函数。B2 执行完成后，B3 将结果转换为 ToolMessage，供下一轮模型推理使用。

## 为什么本文件不应精确匹配

这里的“Agent”出现在第一部分，“工具调用”主要出现在第二部分。两者在文本中被标题、段落和其他词语隔开。本地检索器可以因为两个辅助词都出现而把本文召回，也可以根据覆盖率给出一定分数；但是它不应该把本文误判为一个完整主题为“Agent工具调用”的资料。

## 预期结果

针对 `Agent 工具调用`：本文件通常会显示 `matched_terms` 包含 `agent` 和 `工具调用`，但 `phrase_matched` 应为 `false`。如果它排在 `01_agent_tool_call_exact.md` 前面，说明短语优先排序或评分权重仍然需要调整。
